# OperationOverdose

2D top-down game sandbox built in Python, focused on gameplay systems, real-time interaction, and rapid iteration.

## Overview

- Custom game framework built from scratch with full control over core systems.
- Entity-based architecture supporting players, projectiles, weapons, collision, and interactive objects.
- Real-time update loop designed for responsive gameplay and experimentation.
- Developed as a flexible space for testing ideas and building game systems properly.

## Why

- Game programming is one of the most complete forms of software development because it naturally brings many systems together in one place.
- It provides a platform to work on collision detection, movement, input handling, rendering, state management, entity behaviour, and moment-to-moment interaction within a single project.
- This project exists as a space where I can explore those systems freely, try unusual ideas quickly, and refine them through iteration.
- The focus is not on heavy mathematics or research-driven simulation, but on building good systems in a domain that is broad, technical, and enjoyable to work in.

## What

- Custom vector, transform, and collider systems built from scratch.
- Support for circle, rectangle, and polygon-based collision handling.
- Collision routines including circle-vs-circle, circle-vs-rectangle, and polygon SAT testing.
- Entity and projectile systems designed for extension as mechanics become more complex.
- Weapon behaviour including spread-based firing and controlled collider-ignore logic.
- Movement and gameplay systems that prioritise responsiveness, clarity, and ease of iteration.

## Design Approach

- Built to remain easy to modify as mechanics evolve.
- Systems are written to be reusable, inspectable, and straightforward to extend.
- Refactoring is driven by gameplay needs and system clarity rather than over-engineering.
- The project is experimental in spirit, but the code is still treated seriously.

## Notes

- This project is intentionally a playground for ideas, but not a throwaway one.
- It is where I can explore game programming as a dense systems problem while still keeping the work enjoyable.
- The goal is to write code that is both fun to build and worth reading.
